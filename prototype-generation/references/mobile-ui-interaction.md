# 移动端通用 UI 与交互规范

## 定位

本规范是小程序、H5、原生 APP 等移动端原型的共同底座。它只定义移动端原型的 UI 与交互模式；生成实际 HTML/前端原型时，还必须遵守 `frontend-design` 的通用前端质量要求，包括视觉层级、排版、响应式、动效、可用状态和整体完成度。

当两者冲突时：

- 移动端平台交互习惯优先于桌面网页习惯。
- 业务任务效率优先于装饰性视觉。
- 原型可确认性优先于代码工程完整性。

## 适用场景

- 移动端业务录入、查询、审批、巡检、报数、办理、进度查看。
- 面向一线人员、外勤人员、管理者、客户或会员的手机端业务工具。
- 小程序、移动 H5、原生 APP 的共同页面结构和交互状态。

## 核心原则

- 一屏一个主任务，减少并列复杂操作。
- 优先使用选择、扫码、自动带出、默认值，减少手工输入。
- 核心操作放在拇指容易触达的位置，危险操作远离高频主操作。
- 触摸目标建议不小于 44px，高频按钮不小于 48px。
- 页面层级要浅，列表进入详情、详情发起操作、操作后返回结果。
- 状态反馈必须明确：加载、成功、失败、空数据、无权限、校验错误、部分成功。
- 不把 PC 表格直接搬到手机端；手机端以卡片、列表、分组详情、摘要指标为主。

## 标准页面结构

- 顶部导航：标题、返回、右侧轻操作。
- 内容区：卡片、列表、表单、详情、结果状态。
- 底部区域：TabBar、固定主操作按钮、批量选择工具条或安全确认区。
- 浮层：底部弹层、选择器、确认弹窗、Toast、结果页。

## 常用页面类型

- 登录/授权：账号密码、手机号、微信/企微授权、组织/角色初始化。
- 首页/任务台：待办、提醒、快捷入口、关键指标。
- 列表/检索：搜索、筛选、状态标签、下拉刷新、上拉加载。
- 详情：摘要、状态、关键字段、流程记录、底部主操作。
- 表单/录入：分组字段、选择器、日期、附件、提交反馈。
- 审批/处理：对象摘要、处理意见、通过/驳回、历史记录。
- 消息/通知：待办提醒、业务结果、系统通知。
- 我的/设置：个人资料、角色切换、帮助、退出。

## 列表与卡片

- 列表项必须显示主标题、状态、关键摘要、更新时间或责任人。
- 状态使用文字加颜色标签，不只靠颜色。
- 支持搜索和筛选时，筛选入口要明显，筛选结果为空可一键重置。
- 列表操作优先进入详情后处理；高频轻操作可放在列表项右侧或滑动操作中。

## 表单与录入

- 长表单按业务分组或分步，避免一页堆满字段。
- 必填项明确标识，校验错误就近提示。
- 日期、枚举、人员、组织、项目等优先使用选择器。
- 输入中断或返回时，对未保存内容给出提醒。
- 提交按钮要有 loading/disabled，避免重复提交。
- 提交成功后明确下一步：返回列表、进入详情、继续新增、复制/分享结果。

## 反馈与状态

- Toast：轻量成功、失败、复制成功。
- Dialog：高风险确认、权限说明、不可恢复操作。
- Bottom Sheet：选择、筛选、更多操作、审批意见。
- Result Page：提交成功、提交失败、待审核、无权限。
- Skeleton/Loading：列表、详情、统计加载。
- Empty：无数据时给出下一步，例如新建、重置筛选、联系管理员。

## 导航与返回

- 返回行为必须符合用户预期：表单未保存时提醒，提交后返回上一层或结果页。
- TabBar 页面之间切换不应丢失关键上下文。
- 多层流程要提供明确进度或步骤，例如 1/3、当前节点、下一步。

## 视觉与布局

- 字号以 14-16px 为主，正文不低于 14px，关键数字可放大。
- 卡片圆角建议 12-16px，列表行保持清晰分隔。
- 页面背景使用浅灰，内容卡片用白底，减少大面积高饱和背景。
- 底部固定按钮要避开安全区。
- 不使用桌面式密集表格、复杂多列布局、过小图标按钮。

## 默认科技色与信息层级

移动端原型如果用户未提供品牌色，默认使用科技蓝色系，可参考 PigX 管理后台的蓝色主调，但移动端要更轻、更通透。不得使用偏土、偏灰、偏廉价的棕/橙/脏绿/米色作为默认主视觉。

**品牌与主视觉**

| 用途 | 默认色 | 使用方式 |
|---|---:|---|
| Primary 主色 | `#2E5CF6` | 主按钮、当前 Tab、关键图标、重要数字 |
| Primary Deep 深主色 | `#1D4ED8` | 按下态、深色标题区、强调文字 |
| Primary Light 浅主色 | `#EAF1FF` | 选中底、浅色按钮、信息卡背景 |
| Cyan 辅助科技色 | `#06B6D4` | 渐变、进度、辅助高亮 |
| Indigo 扩展色 | `#6366F1` | 数据维度、次级强调、身份/权限类视觉 |

**页面与容器**

| 用途 | 默认色 |
|---|---:|
| App 外层背景 | `#EEF5FF` 到 `#F8FBFF` 的浅蓝渐变 |
| 页面背景 | `#F6F9FF` |
| 卡片/列表项 | `#FFFFFF` 或 `#F8FBFF` |
| 卡片弱蓝底 | `#F1F7FF` |
| 分割线/边框 | `#D8E6FF` |
| 阴影 | `rgba(46, 92, 246, 0.14)` 到 `rgba(15, 23, 42, 0.08)` |

**文字层级**

| 层级 | 默认色 | 用途 |
|---|---:|---|
| Title | `#0F172A` | 页面标题、列表主标题 |
| Body | `#334155` | 正文、字段值 |
| Secondary | `#64748B` | 摘要、说明、辅助字段 |
| Placeholder | `#94A3B8` | 占位、弱提示 |
| Inverse | `#FFFFFF` | 深色/渐变主按钮上的文字 |

**状态与标签**

| 状态 | 背景 | 文字/图标 | 边框 |
|---|---:|---:|---:|
| 信息/热门/限制 | `#E0F2FE` | `#0369A1` | `#BAE6FD` |
| 成功/已通过 | `#DCFCE7` | `#15803D` | `#BBF7D0` |
| 警告/待处理 | `#FEF3C7` | `#B45309` | `#FDE68A` |
| 危险/失败/冲突 | `#FEE2E2` | `#DC2626` | `#FECACA` |
| 中性/已关闭 | `#F1F5F9` | `#475569` | `#E2E8F0` |

**组合规则**

- 主按钮使用 `#2E5CF6` 或 `#2E5CF6 -> #06B6D4` 渐变，文字为白色。
- 课程/任务/列表卡片默认使用白底或浅蓝底，标题用 `#0F172A`，摘要用 `#64748B`，不要使用黄褐色标签来表达默认高亮。
- “热门、限制、信息提示”等非危险状态优先用蓝/青标签；只有真正失败、冲突、拒绝、危险才使用红色。
- 成功状态可以使用绿色，但只作为语义状态色，不作为 APP 默认主色。
- 页面中同屏高饱和色不超过 3 类：主蓝、辅助青/靛、语义状态色。

## 原型交付要求

移动端 HTML 原型建议使用手机设备框或手机视口展示，至少覆盖：

- 登录/入口。
- 首页/任务台。
- 列表/检索。
- 详情。
- 表单/录入。
- 提交/审批/结果反馈。
- 我的/设置或角色切换。
- 关键空状态、错误状态、无权限状态。
